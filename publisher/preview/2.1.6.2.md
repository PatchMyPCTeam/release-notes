# Publisher Preview Version 2.1.6.2

**Release Date:** 2022-08-26

## Overview

This major preview release introduces Intune package extraction with encryption key storage, granular webhook scoping, ConfigMgr application naming variables, and numerous improvements to multi-tenancy and upload speeds.

> **Note:** Starting with this build, Patch My PC Publisher now requires a minimum of Microsoft .NET Framework 4.6.2.

---

## Features

New capabilities and functionality added in this release.

- **Intune Package Extraction** - Intune package extraction
  - It is now an option to store the encryption keys used to create the Intune package files (.intunewin). This is configurable in the Advanced tab of the Publisher.
  - With the keys stored, you can use the [Intune Application Manager](https://patchmypc.com/intune-application-manager-utility) to download and extract the content of the Patch My PC published Intune applications and updates.
- **Webhook Filtering** - Webhooks can now be granularly scoped based on several criteria listed below. (Requires Enterprise+)
  - *[Requested by users](https://ideas.patchmypc.com/ideas/PATCHMYPC-I-1871)*
  - Webhook Provider: Slack or Teams is now selectable per webhook
  - Notification Level: All, Error, Success
  - Notification type: Update notifications, ConfigMgr app notifications, Intune app notifications, Intune updates notifications, Alert notifications
  - Specific product: Scope a webhook to a specific product
  - Specific tenants: Multi-tenancy webhook scoping
- **ConfigMgr Naming Variables** - Allow variables to be used to customize the ConfigMgr application name and localized application name. Variables available: %VendorName%, %ProductName%, %Version%, %OriginalName%
- **Retention Category Removal** - ConfigMgr application retention now has the option to remove Administrative Categories from retained ConfigMgr applications.
  - *[Requested by users](https://ideas.patchmypc.com/ideas/PATCHMYPC-I-2181)*
- **Security Scope Enforcement** - ConfigMgr security scopes now have the option to enforce the selected scopes. The Publisher will remove all non-selected scopes from the application when Publisher.
  - *[Requested by users](https://ideas.patchmypc.com/ideas/PATCHMYPC-I-2328)*
- **Universal Conflicting Processes** - Allow any product to have Manage Conflicting Processes configured
  - *[Requested by users](https://ideas.patchmypc.com/ideas/PATCHMYPC-I-1699)*

## Improvements

Enhancements to existing functionality.

- **Parallel Intune Sync** - Intune synchronizations will now happen in parallel for multi-tenancy. Up to 20 tenants synchronize at a time for this build.
- **Upload Speed** - Improve the speed of uploading packages to Intune.
- **Email Report Template** - Refactor email report template.
  - The background is now transparent so that it will match the theme of the email client it is opened in.
  - The code used to generate the template has been refactored to simplify future changes.
- **Uninstall String Filtering** - PatchMyPC-Scriptrunner will now factor in the major version filter when available when searching for uninstall strings.
- **Right-Click Tooltips** - Added tooltips to some right-click options that describe why they are disabled in some cases.
- **Assignment Filter Permissions** - Improved the error handling within the Intune Assignments forms regarding permissions for managing Assignment Filters.
- **PowerBI Date Format** - Format the dates using ISO 8601 formatting when doing the Intune App export for PowerBI reporting.
- **WSUS Accessibility** - Improved the accessibility of the WSUS Options form.
- **Exception Messaging** - Improve error messaging and logging for unhandled exceptions.

## Fixes

Bug fixes and issue resolutions.

- **Settings Backup Format** - Fixed a bug where the settings backups were stored in a non-sortable format. This bug was introduced in preview 2.1.6.1.
- **Scan Wizard Filters** - Fixed a bug where the filters were not applied in the scan wizards when filtering the data. This bug was introduced in preview 2.1.6.1.
- **Database Scan Query** - Fixed a bug where the ConfigMgr database scan may throw an exception due to a malformed query. This bug was introduced in preview 2.1.6.1.
- **Failed Logs Copy** - Fixed a bug where the logging option to copy failed logs to a share was not retained. This bug was introduced in preview 2.1.6.1.
- **Additional Files Copy** - Fixed a bug where failing to copy additional files did not cause an Intune product to fail to publish.
- **MCP Localization** - Fixed a bug where localization files for Manage Conflicting Processes may not be copied correctly in some cases.
- **WSUS Scrolling** - Fixed a bug where the WSUS Options window was not scrollable.
- **UI Navigation** - Fixed several UI navigation bugs on the main form.
- **Script Encoding** - Adjusted encoding of detection and requirement scripts to use UTF8. Some scripts were failing to sign with the previous encoding.
- **MCP Organization Name** - Fixed a bug where the Manage Conflicting Process Organization Name was not retained when republishing a ConfigMgr application
